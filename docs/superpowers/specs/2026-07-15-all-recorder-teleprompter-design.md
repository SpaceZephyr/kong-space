# 万物皆可录：提词器与摄像头开关设计

## 目标

在全屏预览页增加一个不会进入导出视频的口播提词器，并允许用户在录制前或录制中关闭、重新开启个人摄像头。关闭摄像头时，麦克风继续录制。

## 交互设计

### 提词器入口

- 在右上角控制组中，将“添加口播脚本”放在“开始录制”按钮之后。
- 点击后打开右上角提词器面板；首次打开显示可编辑文本框。
- 用户写入脚本后可切换到阅读模式。
- 面板支持收起和关闭，脚本文本在当前页面会话中保留。

### 自动滚动

- 阅读模式提供“播放 / 暂停”控制。
- 提供慢、中、快三档速度，默认中速。
- 自动滚动到末尾时停止，并恢复为可再次播放状态。
- 录制过程中可以播放、暂停、调速和手动滚动。

### 摄像头与麦克风

- 摄像头按钮是开启 / 关闭切换，不再在开启后被禁用。
- 首次开启时同时申请摄像头和麦克风权限。
- 关闭摄像头时只停止并移除视频轨道，保留麦克风音频轨道。
- 摄像头关闭时隐藏右下角人物画面，录制 Canvas 不绘制摄像头覆盖层。
- 摄像头关闭后仍允许开始录制，导出内容为演示画面和麦克风声音。
- 重新开启摄像头时只重新申请视频轨道，并复用已有麦克风轨道。
- 返回上传页或离开页面时释放所有媒体轨道。

## 录制隔离

- 提词器和右上角控制区保持在 `stage` 之外，并带 `data-html2canvas-ignore="true"`。
- Canvas 继续仅捕获 `stage` 内容；摄像头开启时由 `drawCameraOverlay` 单独合成。
- 提词器、脚本内容、自动滚动控件、摄像头开关、录制按钮和结束按钮均不得进入导出视频。

## 状态与错误处理

- 没有摄像头权限但麦克风可用时，允许使用纯音频录制。
- 没有麦克风权限时仍允许无声录制，并在状态栏说明当前没有口播音频。
- 录制期间禁止返回文件页，但允许操作提词器和摄像头开关。
- 摄像头重开失败时保留麦克风和当前录制状态，不中断录制。

## 响应式布局

- 桌面端提词器固定在右上角，宽度约 360px，高度不超过视口的 62%。
- 移动端提词器位于底部控制区上方，宽度为视口减去页面安全边距，高度不超过视口的 48%。
- 面板采用紧凑编辑台样式，卡片圆角不超过 8px，避免遮住主要内容区域。

## 验证

- 提词器可编辑、收起、关闭、手动滚动和三档自动滚动。
- 提词器在录制前、录制中均可操作，导出 Canvas 像素中不包含提词器。
- 摄像头开启时视频包含人物画面和麦克风音频。
- 摄像头关闭时视频不含人物画面，但麦克风音频轨道仍存在。
- 摄像头可在录制前和录制中反复开启、关闭。
- 桌面与移动视口无按钮遮挡或横向溢出。
